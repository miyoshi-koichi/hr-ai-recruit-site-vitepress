# 説明会・サイト系
## step1
A. 候補者が「会社を知る」フェーズ
A1. テキスト選択で即質問できる採用サイト
会社説明を読んでいて「ブレスト文化って具体的に何するの？」と思ったら、その場で選択して聞ける。mirai-gikaiのコア機能そのまま。
>> すでにやろうとしてるからやる

A3. 説明の深さ切り替え
mirai-gikaiの「難易度切り替え」を転用。学生向け/経験者向けで説明の粒度を変える。「エンジニア向けに詳しく」を押すと技術スタックの詳細が出てくる、「ざっくり知りたい」を押すと概要だけ、など。
>> これはエンジニアの人はこちらとかで移動するか、エンジニア、デザイナー、その他でわけて分取を切り替えるか。でも切り替える文章そこまでなさそうだから、エンジニアなら知りたいページを増やすだけでいいんじゃないかな

A5. 事業紹介のインタラクティブ探索
ChoiceButtons（選択ボタン）で「どの事業に興味ある？」→ 選んだ事業の詳細をChart/Tableで可視化。「面白プロデュース事業の実績」を円グラフで見せたり、「過去のプロジェクト」をタイムラインで表示したり。テキストの壁ではなく対話的に会社を知れる。
>> 説明見る人がめんどい人ように、質問選びながら知りたいことからチャットで教えてもらうみたいなモード切替できるとよさそう。モード切替が重要で、その人によって理解の仕方や受け止めやすさ変わるから、それを提示して一番いい方法を選べるようにしたい

B3. 選考フローナビゲーター
「エンジニアの選考フローは？」と聞くと TimelineUI で「書類選考 → 技術面接 → カルチャー面接 → 最終」をビジュアル表示。各ステップの詳細も聞ける。
>> 普通に入れたい

C3. Working Memoryで再訪問フォロー
一度サイトを訪れた人が戻ってきたとき、「前回ゲーム事業に興味を持っていましたね。その後新しいプロジェクトが始まりましたよ」と文脈を覚えている。検討中の候補者の背中を押せる。
>> 普通に入れたい

D. 社内（採用チーム側）の改善
D1. 「何を聞かれているか」分析ダッシュボード
nepp-chanのペルソナ分析を転用。候補者がAIに何を質問しているかを集計・可視化。「給与の質問が40%」「リモートワーク可否が30%」→ 採用ページのどこを充実させるべきかがデータでわかる。
>> 普通に入れたい

## step2
D2. AI回答品質の監視
フィードバック機能で候補者が「この回答は役に立った/立たなかった」を評価。Bad評価が多い回答パターンを特定して、ナレッジやプロンプトを改善するサイクルを回せる。

D3. ナレッジ管理の自動化
会社の制度変更、新プロジェクト、新オフィスなどの情報をナレッジ管理UIから更新。ドキュメント変換で社内資料（PDF）を即座にAIの知識に追加。

D4. 採用FAQの自動生成
チャットログから「よく聞かれる質問TOP20」を自動抽出。それを採用サイトのFAQページに自動反映。コンテンツが勝手に充実していく。

D5. Evalで回答品質を担保
nepp-chanのEval機能を転用。「この質問にはこう答えるべき」というテストケースを用意して、プロンプト変更時にAIの回答品質が落ちていないか自動チェック。

E2. AIリファラル支援
社員が知人に会社を紹介するとき、「うちの会社のここが面白いよ」をAIがその人に合わせて説明してくれるリンクを生成。紹介された人がリンクを開くと、AIが1対1で会社説明してくれる。

# AI エントリーシート自動判定とエントリーシート自動作成としてやる
B2. 「自分に合うか」診断
mirai-gikaiのインタビュー機能を転用。AIが5問くらい質問して、カルチャーフィット度をレポート化。「あなたはカヤックの○○な部分と相性が良さそうです」とフィードバック。エントリーへの後押しになる。
C. 候補者が「応募する」フェーズ
C1. AIカジュアル面談 → そのままエントリー
mirai-gikaiのインタビュー機能の最大の転用先。AIと3〜5分話すだけで、志望動機・経験・強みが構造化レポートになり、それがそのままエントリーシート代わりに提出される。「エントリーシート書くのだるい」を解消。
>> 質問でもいいし、他に送ってるエントリーシート送ったらどれくらい合格しそうかA〜E位で判定してくれる。あと、話してるとAIがいい感じのエントリーシート作ってくれるっていうのもやりたいけど、ちゃんとしたもの作るんじゃなくて、コアを掘り下げる、穴埋め式自己分析をやれたらいい

# AI OB訪問としてやる
A4. AI社員ロールプレイ
nepp-chanの「キャラクター性」+ RAGを応用。実際の社員インタビュー記事をナレッジに入れて「エンジニアの○○さんに聞いてみる」のように、社員の視点で答えるAI。本人が答えているわけではないが、社員の声ベースで回答する。
>> OB訪問やってないので、これでやれるといいかもだけど、別にOB訪問的にやるというより、カヤックの情報知りたいだけだから、別にキャラクター入れるよりも情報もらえたらいいんじゃないの？って思ったけど、そこにキャラというフレーバーが入ってることで聞きやすい人が増える可能性もあるかも？テストしてからかなぁ

# フォローアップAI
E4. 選考体験アンケートAI
選考を受けた候補者（合否問わず）に対して、インタビュー機能で「選考体験はどうでしたか？」をAIがヒアリング。構造化レポートで改善点が自動まとまる。
>> これもフォローアップと選考改善としてやりたい！

E3. 内定者フォローAI
内定から入社までの期間、内定者の不安や質問にAIが24時間対応。Working Memoryで「入社前に引っ越しが必要と言ってたけど進捗どう？」みたいな継続的なフォロー。人事の負荷を大幅に削減。
>> これができるとまじいい。できれば、その人の正確から合うキャラクターこっちで選んでクロージングさせたい

# 要検討
E1. 「面白法人ブレストAI」
カヤックの最大の文化である「ブレスト」をAI相手に体験できる。お題を出すとAIが一緒にアイデア出しをしてくれる。候補者が「この会社の考え方、合うかも」と感じる体験型コンテンツ。採用活動であると同時にブランディング。
>> 面白いと持ってもらえるものができるかどうか。アイデアがまだ薄い

# やらない
A2. 「聞きにくいこと」を聞けるAI
「残業多い？」「給料いくら？」「離職率は？」— 人事には聞きにくいことをAIなら気軽に聞ける。匿名認証なので誰が聞いたかわからない。RAGに会社の実態データ（平均残業時間、有給消化率など）を入れておけば正直に答えられる。
>> 面白いかもしれないけれど、そこから情報が流出して、給料が安いとかそういう感じで言われることもあると思うので、身長にしたい

B4. 給与・待遇シミュレーター
「エンジニア5年目だとどのくらい？」に対して、公開可能な範囲でTable/Chartで提示。福利厚生の一覧もテーブルで見やすく。
>> 同上

B. 候補者が「応募を検討する」フェーズ
B1. ポジションマッチングAI
AIが「どんな仕事がしたい？」「得意な技術は？」「働き方の希望は？」を ChoiceButtons で聞いていき、最適なポジションを提案。nepp-chanのマルチエージェント委譲で、職種ごとの専門エージェントに振り分け。
>> 新卒だし、そんなたくさん職種ないから、そもそもどれが合うかな？みたいな迷い方する人少ない

C2. ポートフォリオ対話
nepp-chanのドキュメント変換（PDF → テキスト）を応用。候補者がポートフォリオURLやPDFを送ると、AIがそれを読んで「このプロジェクトについてもう少し教えてください」と深掘り。結果がレポートになる。
>> 流石に難しそうじゃない？できる？精度が悪そう

E5. 採用イベント × AI
会社説明会や採用イベントで参加者がスマホからAIに質問できる。その場で聞きにくいことも匿名で聞ける。イベント後のフォローもWorking Memoryで継続。
>> リアルでやるときはAIいらなくない？その場で聞くでしょ。そのあとに説明会AIを教えればいい
